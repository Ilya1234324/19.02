# README.md: Настройка Git‑репозитория и публикация на GitHub

**Цель:** настроить локальный Git‑репозиторий и загрузить проект на GitHub.

## Ход работы

1. **Инициализация репозитория**

Выполнил команду:
```bash
git init
Результат: Git «запущен» в папке «Задания 19.02». Репозиторий уже существовал — Git переинициализировал его.

Добавление файлов

Выполнил команду:

bash
git add .
Результат: все файлы в папке подготовлены для сохранения (точка . означает «все файлы»).

Первый коммит

Сначала допустил ошибку — ввёл:

bash
git git commit -m "Initial commit"
Git выдал ошибку: git: 'git' is not a git command. Причина — опечатка (двойной git).

Исправил и выполнил правильно:

bash
git commit -m "Initial commit"
Результат: создан первый «снимок» (коммит) файлов.

Переименование ветки

Выполнил команду:

bash
git branch -M main
Результат: ветка переименована с master на main (современный стандарт).

Попытка связать папку с GitHub

Выполнил команду:

bash
git remote add origin https://...
Результат: ошибка — error: remote origin already exists. Оказалось, репозиторий уже связан с другим GitHub‑репозиторием (18.02.2026).

Проверка текущих связей

Выполнил команду:

bash
git remote -v
Результат: увидел, что репозиторий связан с 18.02.2026 (как для fetch, так и для push).

Изменение URL репозитория

Выполнил команду:

bash
git remote set-url origin https://...
Результат: обновил ссылку на новый GitHub‑репозиторий (19.02). Теперь Git будет отправлять файлы туда.

Отправка файлов на GitHub

Выполнил команду:

bash
git push origin main
Результат: файлы отправлены на GitHub в репозиторий 19.02. Git:

посчитал объекты (Enumerating objects);

сжал их (Compressing objects);

отправил (Writing objects).

Добавление дополнительного репозитория

Выполнил команду:

bash
git remote add new-origin https://...
Результат: добавил ещё одну «связь» с GitHub — теперь есть резервный (или дополнительный) репозиторий.

Настройка «автоматического» push

Выполнил команду:

bash
git push -u origin main
Результат:

закрепил связь между локальной веткой main и веткой на GitHub;

настроил «автоматический» push (теперь достаточно просто git push);

Git вывел: branch 'main' set up to track 'origin/main';

финальный статус: Everything up-to-date.
